The Rooms.cs class handles the XML loading inside a static constructor, which makes it 
hard to debug if the file is missing and prevents you from easily loading different levels. 
You should move this logic into a separate method and use int.TryParse to prevent the game 
from crashing if the data has typos. It is also important to verify that the tile data actually
exists before creating a room to avoid errors later on. Making the class non-static and adding 
these checks would make your code much more flexible and reliable.
